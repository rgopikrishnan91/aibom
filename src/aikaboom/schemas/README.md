# SPDX 3.0.1 Schema Files

Official validation artifacts for SPDX 3.0.1, used by AIkaBoOM's emitter and validator.

## Files

- **spdx-json-schema.json** - Validation subset of the official JSON Schema (generated by
  `shacl2code`). Contains type names, required properties, and property name conventions.
  Full schema: `https://spdx.org/schema/3.0.1/spdx-json-schema.json`

- **spdx-context.jsonld** - JSON-LD context mapping short property names to full SPDX URIs.
  This is the Rosetta Stone for understanding property naming conventions:
  - Core properties have no prefix: `name`, `creationInfo`, `suppliedBy`
  - AI properties use `ai_` prefix: `ai_domain`, `ai_typeOfModel`
  - Software properties use `software_` prefix: `software_downloadLocation`
  - Dataset properties use `dataset_` prefix: `dataset_datasetType`
  Full context: `https://spdx.org/rdf/3.0.1/spdx-context.jsonld`

- **spdx-property-map.json** - AIkaBoOM-specific mapping from our internal field names
  to the official SPDX 3.0.1 JSON-LD property names. Used by the emitter.
